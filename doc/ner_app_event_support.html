NER-APP Event Support Model

Overview

The intention is for the NER-App to supplement and improve an inefficient paper-based process that is used for tracking event data. This begins with importing data from BikeReg, where half of an event's riders will pre-register. This data will be the basis for an offline form that will be used for tracking rider progress during an event. On the day of the event, the offline form will also accept data for day-of-registration participants and will then be used for recording times and status for riders through the event. At the end of the event, the data in the form will be imported back into the NER-app. This updated data will then be available to riders looking to review their historical results.

BikeReg Import

Importing data from BikeReg will, by default, take the form of parsing a CSV file exported from bikereg. The import process shall read data from this CSV into a staging location where an admin will review the inbound data, confirm matches to existing data, confirm new records, and reject invalid data. The intention for the import is to:

  1. add new User records to the application where individuals do not exist.
  2. add additional data to a User's record if that data was previously null.
  3. otherwise match inbound data to existing records based on the following criteria (TBD):
    1. email address
    2. first name
    3. last name
    4. date of birth
  4. spawn additional Registrations and RouteControlStamps for all registered riders

If data discrepancies exist between user data BikeReg and NER-App, NER-App should be considered the source of truth.

Offline Form

Data imported from BikeReg should be exported to a form that can be manipulated or edited offline. Currently, the preference for this form is as an Excel spreadsheet. The form will capture:

  1. Rider Registration Data
    1. email address
    2. first name
    3. last name
    4. phone number
    5. date of birth
    6. address data
    7. NER number/status
    8. RUSA number/status
    9. emergency contact
  2. Payment provided (if not pre-registered)
    1. Payment amount
    2. Payment method (cash/check)
  3. Ride Status
    • Completed
    • DNS
    • DNF
    • DQ
    • Other
  4. All associated route card stamps

At the end of an event, the form will export its data in a fashion that can then be imported into NER-App.

Offline Form Import

Similar to BikeReg Import, the data of the Offline Form should be imported into a staging location in the NER-App where an admin will review the inbound data, confirm matches to existing data for day-of-registrants, confirm new records and reject invalid data.

Volunteer Tracking

Separate from day of registrations, the admin or brevet organizer should have the ability to assign NER-APP users as volunteers for an event. Volunteers can be assigned to any one of the following roles:

  1. Arrowing
  2. Pre-ride
  3. Controle staff

Volunteers can have more than one role for an event. A pre-ride volunteer should also be treated as a rider if they are riding for mileage credit, and so their data should be included in the ride results.
